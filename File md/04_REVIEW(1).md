# AI Self Review Protocol

## Objective

Never consider work complete until it has been reviewed like a senior engineer.

---

# Phase 1 — Requirement Validation

- Does the solution solve the user's actual problem?
- Were all requirements implemented?
- Were assumptions avoided?
- Are missing requirements documented?

---

# Phase 2 — UI Review

Review:

- Typography
- Spacing
- Visual hierarchy
- Grid consistency
- Alignment
- Contrast
- White space
- Icons
- Responsiveness
- Dark mode

If any section feels generic, redesign it.

---

# Phase 3 — UX Review

Check:

- Navigation
- User flow
- Discoverability
- Empty states
- Loading states
- Error states
- Success feedback

---

# Phase 4 — Code Review

- Meaningful naming
- No duplicated logic
- Reusable components
- Small focused functions
- Strict typing
- Error handling
- Clean folder structure

---

# Phase 5 — Performance

Verify:

- Images optimized
- Lazy loading
- Dynamic imports
- Bundle size
- Rendering strategy
- Caching

---

# Phase 6 — Accessibility

- Semantic HTML
- Keyboard navigation
- Focus states
- Color contrast
- Alt text
- Labels
- ARIA where needed

---

# Phase 7 — SEO

- Title
- Description
- Open Graph
- Structured headings
- Canonical URL
- Sitemap
- Robots

---

# Phase 8 — Production Checklist

Before finishing:

[ ] Responsive
[ ] Accessible
[ ] Performant
[ ] Secure
[ ] Reusable
[ ] Production Ready

If any checkbox is false:

Continue improving.

Never stop at "good enough".

End of Review Protocol.
