# 29_UI_PATTERNS.md

# UI Patterns Library

## Objective

Use proven UI patterns instead of inventing layouts.

---

# Navigation

Portfolio
- Sticky top navigation
- Logo
- Section links
- Primary CTA

Dashboard
- Sidebar
- Topbar
- Breadcrumb
- Search

---

# Hero

Must contain:

- Strong headline
- Value proposition
- Primary CTA
- Secondary CTA
- Supporting visual

Never use oversized empty heroes.

---

# Section Pattern

Title

↓

Short description

↓

Content

↓

Optional CTA

Maintain consistent spacing.

---

# Cards

Use only when grouping related information.

Every card should contain:

- Title
- Description
- Optional action
- Consistent padding

Avoid decorative cards.

---

# Timeline

Recommended for:

- Experience
- Education
- Career
- Certifications

Always show chronological order.

---

# Metrics

Only display real metrics.

Avoid fake statistics.

---

# Testimonials

Only include verified testimonials.

If unavailable, remove the section.

---

# Forms

Every form should include:

- Labels
- Validation
- Error state
- Success state
- Loading state

---

# Empty States

Every empty state should answer:

- Why is this empty?
- What should the user do next?

---

# Loading States

Prefer:

- Skeleton loaders
- Progressive loading

Avoid infinite spinners.

---

# Responsive Patterns

Mobile:
Single column

Tablet:
Two-column where appropriate

Desktop:
Multi-column grid

---

# Accessibility

Every pattern must support:

- Keyboard navigation
- Screen readers
- Visible focus
- Semantic HTML

---

# Final Rule

Choose familiar patterns that reduce cognitive load.

Consistency is more valuable than novelty.

End of UI Patterns.
