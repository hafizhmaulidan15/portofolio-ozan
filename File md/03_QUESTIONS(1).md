# 03_QUESTIONS.md

# AI Discovery Framework

## Rule

Never begin implementation before understanding the problem.

Target confidence: **95%+**

---

# Product

- What is the product?
- Why is it being built?
- What problem does it solve?
- What is the MVP?
- What defines success?

---

# Users

- Who are the primary users?
- What are their goals?
- What frustrates them today?
- Desktop, mobile, or both?

---

# Brand

- Brand personality
- Tone of voice
- Color palette
- Typography
- Premium or playful?
- Competitors
- Websites you admire

---

# Content

- Do you already have copy?
- Logo?
- Images?
- Icons?
- SEO keywords?
- Social links?

---

# Design

- Preferred layout
- Dark/light
- Animation level
- Grid style
- Card style
- Navigation
- Footer

---

# Features

- Authentication
- Dashboard
- CMS
- Blog
- Search
- Filters
- Payments
- Uploads
- Notifications
- AI
- Export

---

# Technical

- Framework
- Backend
- Database
- ORM
- Storage
- Deployment
- CI/CD
- Analytics

---

# Performance

- Lighthouse target
- SEO target
- Accessibility target
- Browser support

---

# Future

- Expected growth
- Multi-language?
- Mobile app?
- Admin panel?
- API?
- Integrations?

---

# AI Behavior

If any answer is missing:

DO NOT ASSUME.

Ask follow-up questions until requirements are complete.

If requirements conflict:

Explain why.

Recommend a better solution.

End of file.
