# 08_LAYOUT.md

# Layout System

## Philosophy

Layout creates understanding before design creates emotion.

A great layout is invisible.

---

# Core Principles

- Content first
- Grid first
- Consistent spacing
- Predictable rhythm
- Responsive by default

---

# Page Structure

Navbar
↓

Hero
↓

Value Proposition
↓

Content Sections
↓

Call To Action
↓

Footer

Avoid random section ordering.

---

# Container Width

Recommended:

- Mobile: 100%
- Tablet: 720px
- Desktop: 1200–1440px max

Never let text span the entire screen.

---

# Spacing Scale

4
8
12
16
24
32

Use the same spacing scale everywhere.

---

# Section Rhythm

Small section gap:
64px

Normal:
80px

Large:
120px

---

# Grid

Prefer:

- 12-column desktop
- 8-column tablet
- 4-column mobile

Align everything to the grid.

---

# Cards

Cards are not decoration.

Every card must have a purpose.

Avoid excessive borders and shadows.

---

# Responsive Rules

Desktop first thinking is allowed,
but implementation must work on:

- Mobile
- Tablet
- Laptop
- Desktop
- Ultrawide

---

# White Space

White space is a feature.

Do not fill empty space without purpose.

---

# Visual Flow

Every page should naturally guide users:

Headline
↓

Explanation
↓

Evidence
↓

Action

---

# Anti Patterns

Avoid:

- Crowded layouts
- Giant empty hero
- Uneven spacing
- Misaligned components
- Nested cards everywhere

---

# Goal

The interface should feel structured, calm, intentional, and easy to scan.

End of Layout Guide.
