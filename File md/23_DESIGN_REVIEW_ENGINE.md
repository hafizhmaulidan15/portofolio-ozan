# 23_DESIGN_REVIEW_ENGINE.md

# Design Review Engine

## Mission

Review every UI like a Principal Product Designer before delivery.

Never accept the first version as the final version.

---

# Review Order

1. Information hierarchy
2. Layout
3. Typography
4. Color
5. Components
6. Motion
7. Accessibility
8. Responsiveness
9. Performance
10. Brand consistency

---

# Information Hierarchy

Ask:

- Is the most important content visible first?
- Is the CTA obvious?
- Can users scan the page in under 10 seconds?
- Is every section necessary?

Remove unnecessary content.

---

# Layout

Verify:

- Consistent spacing
- Grid alignment
- Predictable flow
- Balanced whitespace
- Comfortable reading width

Avoid visual clutter.

---

# Typography

Review:

- Heading scale
- Font weights
- Line height
- Readability
- Paragraph length

Typography should communicate before decoration.

---

# Components

Check:

- Consistent sizing
- Consistent radius
- Reusable patterns
- Interactive states
- Empty, loading, error states

---

# Motion

Motion should:

- Guide attention
- Confirm interactions
- Preserve context

Avoid distracting animations.

---

# Accessibility

Validate:

- Keyboard navigation
- Focus states
- Contrast
- Semantic HTML
- Alt text

---

# Responsive Design

Review on:

- Mobile
- Tablet
- Laptop
- Desktop

No layout should break.

---

# Brand

Confirm that the design reflects the intended personality.

Examples:

Portfolio:
Professional.

Law Firm:
Authoritative.

Startup:
Friendly.

---

# Final Score

Score each category from 1–10.

If any category is below 8:

Refine the design before delivering.

---

# Final Rule

Never ship a design you would not proudly use as your own portfolio.

End of Design Review Engine.
