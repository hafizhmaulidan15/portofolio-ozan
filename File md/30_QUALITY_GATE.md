# Quality Gate

## Purpose

No project may be considered complete until it passes every quality gate.

---

# Gate 1 — Requirements

- All requested features implemented
- No undocumented assumptions
- User goals satisfied

---

# Gate 2 — Design

- Clear hierarchy
- Consistent spacing
- Professional typography
- Balanced layout
- Consistent components

Score: 8/10 minimum.

---

# Gate 3 — UX

Verify:

- Clear navigation
- Intuitive interactions
- Helpful feedback
- Empty states
- Error states
- Success states

---

# Gate 4 — Accessibility

Must support:

- Keyboard navigation
- Visible focus
- Semantic HTML
- Alt text
- Color contrast

---

# Gate 5 — Performance

- Optimized images
- Optimized fonts
- Minimal client JS
- Lazy loading
- Stable layout

Target Lighthouse: 90+

---

# Gate 6 — Code Quality

- Strict typing
- No duplicated logic
- Reusable components
- Clean architecture
- Predictable naming

---

# Gate 7 — SEO

- Metadata
- Open Graph
- Sitemap
- Robots
- Canonical URL
- Structured headings

---

# Gate 8 — Security

- No exposed secrets
- Validated input
- Safe dependencies
- Friendly error messages

---

# Final Self Review

Answer honestly:

1. Would I deploy this today?
2. Would I be proud to show this to a senior engineer?
3. Is there a simpler solution?
4. Is every dependency justified?
5. Does every section have a purpose?

If any answer is "No":

Stop.

Refactor.

Review again.

---

# Ship Checklist

[ ] Requirements complete
[ ] Design approved
[ ] UX approved
[ ] Accessibility approved
[ ] Performance approved
[ ] SEO approved
[ ] Security approved
[ ] Code reviewed
[ ] Production ready

End of Quality Gate.
