# 06_DESIGN_SYSTEM.md

# Design System

## Philosophy

Less, but better.

Every pixel must have a purpose.

Design should communicate before it decorates.

---

# Core Principles

- Typography first
- Content first
- Consistent spacing
- Clear hierarchy
- Accessibility by default
- Responsive by default

---

# Design DNA

Keywords:

- Professional
- Premium
- Minimal
- Editorial
- Engineering-focused
- Timeless

Avoid generic AI-generated layouts.

---

# Visual Hierarchy

Priority order:

1. Headline
2. Supporting text
3. Primary action
4. Secondary content
5. Metadata

---

# Spacing Scale

Use only:

4
8
12
16
24
32
48
64
80
96
120

Never invent random spacing values.

---

# Color Rules

- Neutral foundation
- One accent color
- Semantic colors only
- High contrast
- Accessible combinations

---

# Typography Rules

- Maximum two font families
- Strong heading hierarchy
- Readable paragraph width
- Consistent line height

---

# Components

Every component must be:

- Reusable
- Predictable
- Accessible
- Composable
- Responsive

---

# Motion

Motion should:

- Guide attention
- Confirm interactions
- Never distract
- Respect reduced-motion

---

# Design Checklist

Before shipping:

[ ] Strong hierarchy
[ ] Consistent spacing
[ ] Readable typography
[ ] Accessible contrast
[ ] Responsive layout
[ ] Premium visual quality
[ ] Consistent components

---

# Final Rule

If removing an element improves clarity,
remove it.

End of Design System.
