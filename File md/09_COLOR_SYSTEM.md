# 09_COLOR_SYSTEM.md

# Color System

## Philosophy

Color communicates meaning before decoration.

Use color intentionally.

---

# Core Principles

- Neutral foundation
- Single primary accent
- High contrast
- Accessible combinations
- Consistent semantics

---

# Foundation

Background
- Primary surface
- Secondary surface
- Elevated surface

Text
- Primary
- Secondary
- Muted
- Disabled

Border
- Subtle
- Strong
- Focus

---

# Semantic Colors

Success
- Positive feedback
- Completed actions

Warning
- Attention required
- Recoverable issues

Error
- Validation failures
- System failures

Info
- Neutral guidance
- Tips
- Status updates

Never use semantic colors as branding colors.

---

# Accent Color

Choose one primary accent.

Rules:

- Use for CTA
- Active navigation
- Important highlights
- Interactive states

Avoid using accent color everywhere.

---

# Dark Mode

Dark mode is preferred.

Guidelines:

- Avoid pure black (#000000)
- Use layered surfaces
- Maintain strong text contrast
- Reduce eye strain

---

# Contrast

Meet accessibility standards.

- Text must remain readable.
- Never rely on color alone.
- Pair color with icons, labels, or patterns when needed.

---

# Hover & Focus

Hover:
- Slight brightness
- Subtle elevation

Focus:
- Clear visible outline
- Keyboard friendly

Active:
- Strong but consistent feedback

---

# Anti Patterns

Avoid:

- Rainbow palettes
- Neon colors
- Random gradients
- Multiple accent colors
- Low-contrast text
- Decorative color usage

---

# Brand Consistency

Every screen should feel like part of the same product.

Users should recognize the interface without seeing the logo.

---

# Goal

Color should improve clarity, hierarchy, accessibility, and brand recognition—not become the main attraction.

End of Color System.
