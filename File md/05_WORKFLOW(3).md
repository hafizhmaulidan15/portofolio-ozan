# OpenCode Standard Workflow

## Philosophy

Never jump directly into coding.

Every project follows the same engineering workflow.

---

# Phase 1 — Discovery

Objectives:

- Understand business goals
- Understand users
- Understand constraints
- Understand success metrics

Tasks:

- Read project documentation
- Read AI framework
- Classify project
- Ask questions until confidence ≥95%

Deliverables:

- Problem Statement
- Requirements Summary

---

# Phase 2 — Planning

Tasks:

- Create Product Brief
- Define Features
- Create Information Architecture
- Choose Architecture
- Plan Folder Structure

Deliverables:

- PRD
- Architecture Plan
- Component List

---

# Phase 3 — Design

Tasks:

- Define Layout
- Typography
- Color System
- Motion
- Component System

Deliverables:

- Design Direction
- UI Hierarchy

---

# Phase 4 — Build

Tasks:

- Create reusable components
- Build layouts
- Implement features
- Handle loading, empty, success, and error states

Rules:

- Small commits
- Reusable code
- Strict TypeScript

---

# Phase 5 — Review

Review:

- Requirements
- UX
- Accessibility
- Performance
- SEO
- Security
- Code Quality

Refactor if necessary.

---

# Phase 6 — Deliver

Provide:

- Summary
- Architecture decisions
- Trade-offs
- Future improvements

Never ship unfinished work.

End of Workflow.
